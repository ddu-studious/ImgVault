## 一、概述

### 1.1 背景
ImgVault 目前支持单图上传、存储、缩放、格式转换等能力。用户在实际使用中经常需要将多张图片和文字组合到一张图片上，生成海报、拼图、产品展示图等。此外，去除图片水印也是高频需求。

### 1.2 目标
在 ImgVault 中新增三大核心能力：
- **图片合成 API** — 多图+文字组合为一张图，支持模板和自由画布
- **可视化合成编辑器** — 基于 Fabric.js 的拖拽式所见即所得编辑器，替代手动 JSON 参数输入
- **图片去水印** — OpenCV Inpainting (轻量) + IOPaint Docker (AI 高质量) 分层架构

### 1.3 用户价值
- 电商场景：快速生成产品组合图
- 内容运营：可视化拖拽制作社交媒体海报
- 图片修复：去除图片水印/Logo/多余文字
- 个人使用：图片拼接、添加文字注释

### 1.4 技术约束
- **全部免费技术** — 不使用任何付费 API 或商业 SDK
- **Java 技术栈为主** — Graphics2D + OpenCV Java Binding
- **AI 能力自部署** — IOPaint (Apache-2.0) Docker 部署

## 二、功能需求

### F1: 自由画布合成 API

**描述**: 用户提交合成指令（JSON），指定画布尺寸、背景色，以及多个图片/文字图层，后端按指令合成并返回结果。

**API 设计**:

```
POST /imgvault/api/v1/compose
Content-Type: application/json
```

**请求体**:
```json
{
  "canvas": {
    "width": 1920,
    "height": 1080,
    "backgroundColor": "#FFFFFF"
  },
  "layers": [
    {
      "type": "image",
      "imageId": 123,
      "x": 0,
      "y": 0,
      "width": 960,
      "height": 1080,
      "opacity": 1.0,
      "fit": "cover",
      "borderRadius": 0
    },
    {
      "type": "image",
      "imageId": 456,
      "x": 960,
      "y": 0,
      "width": 960,
      "height": 540,
      "opacity": 1.0,
      "fit": "cover"
    },
    {
      "type": "text",
      "content": "春季新品上市",
      "x": 100,
      "y": 900,
      "fontSize": 64,
      "fontFamily": "NotoSansCJK",
      "fontWeight": "bold",
      "color": "#333333",
      "maxWidth": 800,
      "lineHeight": 1.5,
      "textAlign": "left",
      "shadow": {
        "color": "rgba(0,0,0,0.3)",
        "offsetX": 2,
        "offsetY": 2,
        "blur": 4
      }
    }
  ],
  "output": {
    "format": "png",
    "quality": 90
  }
}
```

**响应**:
```json
{
  "code": 200,
  "data": {
    "imageId": 789,
    "imageUuid": "abc-123-...",
    "downloadUrl": "https://xxx/imgvault/storage/composed/2026/03/08/abc-123.png",
    "width": 1920,
    "height": 1080,
    "fileSize": 524288,
    "format": "png"
  }
}
```

**图层类型**:

| 类型 | 属性 | 说明 |
|------|------|------|
| `image` | imageId, imageUrl, x, y, width, height, opacity, fit, borderRadius, rotation | 图片图层 |
| `text` | content, x, y, fontSize, fontFamily, fontWeight, color, maxWidth, lineHeight, textAlign, shadow | 文字图层 |
| `shape` | shape(rect/circle/line), x, y, width, height, color, borderColor, borderWidth, opacity | 形状图层 |

### F2: 内置模板合成

**描述**: 提供常用合成模板，用户只需指定图片和文字，无需手动计算坐标。

**API 设计**:
```
POST /imgvault/api/v1/compose/template
```

**请求体**:
```json
{
  "template": "grid-2x2",
  "images": [123, 456, 789, 101],
  "text": {
    "title": "四季风光",
    "subtitle": "2026 精选"
  },
  "output": {
    "format": "jpeg",
    "quality": 85,
    "width": 1920
  }
}
```

**内置模板**:

| 模板 ID | 名称 | 说明 |
|---------|------|------|
| `grid-2x2` | 2×2 网格 | 4 张图等分排列 |
| `grid-3x3` | 3×3 网格 | 9 张图等分排列 |
| `grid-1-2` | 1+2 布局 | 左侧大图 + 右侧 2 张小图 |
| `grid-1-3` | 1+3 布局 | 顶部大图 + 底部 3 张小图 |
| `horizontal` | 横向拼接 | N 张图水平排列 |
| `vertical` | 纵向拼接 | N 张图垂直排列 |
| `poster` | 海报 | 背景图 + 标题 + 副标题 |
| `product-card` | 产品卡 | 产品图 + 名称 + 描述 + 价格 |
| `comparison` | 对比图 | 左右两图对比 + 标签 |
| `collage` | 自由拼贴 | N 张图随机角度拼贴 |

### F3: 模板预览

```
GET /imgvault/api/v1/compose/templates
```

返回所有可用模板的列表和预览缩略图。

### F4: 合成历史

合成结果自动存入 `img_image` 表，标记 `source_type = 'composed'`，可通过现有图片列表 API 查询。

### F5: 可视化合成编辑器 (新增)

**描述**: 基于 Fabric.js 的前端所见即所得编辑器，用户通过拖拽、缩放直接编排图层。

**技术方案**: Fabric.js (MIT, 29k+ Stars) + 原生 HTML/JS

**核心功能**:

| 功能 | 说明 |
|------|------|
| 画布设置 | 自定义宽高、背景色 |
| 图片图层 | 从图库选图加载到画布，拖拽移动、缩放、旋转 |
| 文字图层 | 添加文字，实时编辑内容、字体、颜色、大小 |
| 形状图层 | 矩形、圆形、线条，自定义填充色和边框 |
| 图层管理 | 图层列表面板，上移/下移/删除/锁定 |
| 属性面板 | 选中元素后显示位置/尺寸/旋转/透明度等精确数值编辑 |
| 对齐辅助 | Snap to Grid / Smart Guides 对齐辅助线 |
| 模板加载 | 从模板列表一键加载预设布局到画布 |
| 导出合成 | 画布序列化为 Compose API JSON 格式，调用后端生成高质量合成图 |
| Undo/Redo | 撤销/重做操作历史 |

**交互流程**:
1. 用户进入合成编辑器页面
2. 设置画布尺寸，或选择模板快速开始
3. 从图库选图添加到画布（弹出图库选择器）
4. 拖拽调整位置、缩放大小、旋转角度
5. 添加文字/形状图层
6. 点击"生成合成图"，前端序列化画布状态为 API JSON 提交后端
7. 后端 Graphics2D 渲染高质量合成图，存储到 MinIO
8. 返回结果，显示下载链接

### F6: 图片去水印 (新增)

**描述**: 提供图片去水印能力，分轻量级 (OpenCV) 和 AI 高质量 (IOPaint) 两层。

**API 设计**:

#### 6.1 去水印 API

```
POST /imgvault/api/v1/watermark/remove
Content-Type: multipart/form-data
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `imageId` | Long | 已有图片 ID (与 file 二选一) |
| `file` | MultipartFile | 上传图片文件 (与 imageId 二选一) |
| `maskData` | String | Base64 编码的 Mask 图片 (白色=水印区域) |
| `engine` | String | 处理引擎: `opencv` (默认) / `iopaint` |

**响应**:
```json
{
  "code": 200,
  "data": {
    "imageId": 790,
    "imageUuid": "def-456-...",
    "downloadUrl": "https://xxx/imgvault/storage/watermark-removed/...",
    "width": 1920,
    "height": 1080,
    "fileSize": 524288,
    "format": "png",
    "engine": "opencv"
  }
}
```

#### 6.2 前端可视化去水印

**技术方案**: 复用 Fabric.js Canvas

**交互流程**:
1. 用户上传/选择图片，加载到画布
2. 使用画笔工具/矩形工具标记水印区域 (红色半透明叠加)
3. 画笔标记区域自动转换为 Mask 图片 (白底黑底)
4. 选择处理引擎 (OpenCV 快速 / IOPaint AI 高质量)
5. 提交后端处理
6. 预览去水印效果，支持对比查看
7. 确认保存/下载

## 四、技术方案

### 4.1 架构

```
ComposeController (API层)      WatermarkController (API层)
    │                               │
ComposeAppService (应用层)     WatermarkAppService (应用层)
    │                               │
    ├── TemplateEngine              ├── OpenCvWatermarkService    # OpenCV Inpainting
    ├── ImageComposeService         └── IoPaintWatermarkService   # IOPaint HTTP 代理
    │      ├── ImageLayerRenderer
    │      ├── TextLayerRenderer
    │      └── ShapeLayerRenderer
    ├── MinioStorageService
    └── ImageRepository
```

### 4.2 核心实现

- 使用 Java `BufferedImage` + `Graphics2D` 实现画布合成
- 图层按 `layers` 数组顺序从下到上绘制（后面的图层覆盖前面）
- 图片图层通过 `MinioStorageService` 从 MinIO 拉取源图
- 文字图层使用 `Font` + `FontMetrics` 计算布局，支持自动换行
- 合成结果写入 MinIO `composed/` 目录，记录到 `img_image` 表
- 去水印使用 OpenCV Java Binding (`org.bytedeco:opencv-platform`)
- IOPaint 通过 Docker 部署，Java 通过 HTTP API 调用

### 4.3 字体方案

- 内置 Noto Sans CJK 字体文件 (约 16MB)
- 放置在 `resources/fonts/` 目录
- 启动时注册到 `GraphicsEnvironment`
- 支持 bold、italic 变体

### 4.4 前端可视化编辑器

- 基于 Fabric.js (CDN 引入，无需构建工具)
- 合成编辑器: `frontend/compose.html` + `frontend/compose.js`
- 去水印工具: `frontend/watermark.html` + `frontend/watermark.js`
- 与现有原生 HTML/JS 前端风格统一

### 4.5 去水印分层架构

```
请求 → 策略路由
        ├── engine=opencv   → OpenCV Inpainting (Java 原生, 快速)
        └── engine=iopaint  → IOPaint Docker (AI 高质量, 复杂水印)
```

## 六、验收标准

1. `POST /compose` 接口可正常合成多图+文字
2. 支持至少 4 种内置模板
3. 中文文字渲染正常（无乱码、无截断）
4. 合成结果自动存入 MinIO 并返回访问链接
5. 合成图片可在前端/Admin 端正常查看
6. **可视化编辑器**: 支持拖拽图片/文字到画布，调整位置大小，导出生成合成图
7. **去水印 (OpenCV)**: 提交图片+Mask，返回去水印后的图片
8. **去水印 (IOPaint)**: IOPaint Docker 正常启动，API 调用返回高质量去水印结果
9. **前端去水印**: 画笔标记水印区域，提交处理，预览对比效果
10. **运维脚本**: start.sh/stop.sh 正常管理所有服务 (含 IOPaint)
